## 도큐사우르스에 있는 주스탠드

## 개요
- Zustand는 작고 빠르며 확장 가능한 React 프로젝트에서 사용하는 상태 관리(Store) 라이브러리

### 스토어
- 스토어는 애플리케이션의 여러 상태(State)를 중앙에서 관리하는 패턴을 말함
- 이를 통해 컴포넌트 간 데이터를 쉽게 공유하고 데이터 변경을 감지해 자동으로 렌더링
- 대부분의 애플리케이션은 다음과 같이 최상위 컴포넌트(App)을 기준으로 하는 여러 하위 컴포넌트(A1, B1, B2)등으로 구성됨
- 만약 컴포넌트 간 공유해야 하는 데이터가 있다면, 기본적으로 부모와 자식 컴포넌트 간 데이터 전달이 가능함
- 그리고 이렇게 데이터를 전달하는 것을 Props(Emits) 방식이라고 함
- 이러한 방식은 부모와 자식 관계 이상의 중첩된 컴포넌트 구조에서 불필요하게 데이터를 취급하는 중간 단계의 컴포넌트가 생기고, 그만큼 컴포넌트 간 결합도가 높아지며 유지/보수 를 매우 어렵게 만듬
- 만약 App-B1-B2-B3.1, B3.2로 구성되어있는 컴포넌트가 있다는 가정에 app에서 전달한 props를 b3.1로 전달하려면 B1, B2를 거쳐야 함
- 이런 상황을 속성이 여러 컴포넌트를 관통하는것 같다고 하여 Prop Drilling이라고 부름

### 스토어의 사용
- 스토어의 사용은 컴포넌트간 공유할 데이터를 중앙에서 관리할 수 있다.
- 이 방식은 데이터를 전달하는 중간 단계 컴포넌트가 필요치 않아 컴포넌트간 결합도를 낮추고 유지/보수를 쉽게 만듬

### 기본 사용
- create 함수로 스토어를 생성함
- create 함수의 콜백은 set, get 매개변수를 가지며, 이를 통해 상태를 변경하거나 조회할 수 있음
- create 함수의 콜백이 반환하는 객체에서 속성은 상태(state)이고, 메소드는 액션(action)이라고 부름
- create 함수 호출에서 반환하는 스토어 훅 (Hook)은 useCounterStore와 같이 use 접두사와 Store 접미사로 명명해 각 컴포넌트에서 사용할 수 있다.
- 변경할 상태를 속성으로 포함한 객체를 콜백에서 반환해야 함

```
import { create } from 'zustand'

interface CounterState {
    count : number,
    increment : () => void;
    defrement : () => void;
    reset : () => void;
}
export const useBasicCounter = create<CounterState>((set, get) => ({
    count: 0,
    increment: () => set((state) => ({count: state.count + 1})),
    decrement: () => {
        const state = get();
        const { count } = state;
        set({count : count-1});
    },
    reset: () => set({ count : 0 }),
}));

```

### 컴포넌트에서의 호출
- 컴포넌트에서 스토어 훅(useBasicCounterStore)을 가져와 호출하면, 상태와 액션을 가진 객체를 얻을 수 있음
- 또한 상태(count)는 반응형이기 때문에 상태가 변경되면 컴포넌트가 다시 렌더링됨
```
import { useBasicCounterStore }  from '@/store/counter';
import { useShallow } from 'zustand/react/shallow';

// 구조 분하로 가져오면 상태(count) 변경 시 Action을 사용하는 컴포넌트로 리렌더링됨(사용 x)
const { count, increment, decrement, reset } = useBasicCounterStore();

// 방법1. 상태를 개별적으로 불러오고, Action은 getState()로 사용
const count = useBasicCounter((state) => state.count);
const { increment, decrement, reset } = useBasicCounterStore.getState();

// 방법2 useShallow 사용
// 상태에 해당하는 것만 리렌더링 됨. (Action은 리렌더링 안됨)
const {count, increment, decrement, reset} = useBasicCounterStore(
    useShallow((state) => ({
        count: state.count,
        increment: state.increment,
        decrement: state.decrement,
        reset: state.reset,
    })),
);

<div style={{margin: 20, flex:1}}>
    <h1></h1>
    <div>
        <button onClick={increment}>증가</button>
        <button onClick={decrement}>감소</button>
        <button onClick={reset}>리셋</button>
    </div>
</div>


```

### 미들웨어 사용
- Zustand에서 지원하는 미들웨어가 있음
- combine (여러 상태를 하나로 합치는 미들웨어 - 상태병합이 자동으로 처리, 타입 추론 지원)
- immer(상태 불변성을 자동으로 관리 - 상태를 직접 변경, 중첩된 객체 변경 시 유용함)
- subscriveWithSelector(상태 변화를 감지하는 미들웨어, Listener를 등록하여 특정 로직 실행 가능)
- devtools (Redux DevTools와 연결하여 상태 변경을 추적할 수 있도록 지원)
- persist( 스토리지 저장 )
```
export const useDevToolsCounterStore = create<CounterState>()(
    devtools(
        persist(
            immer(
                comvine({count : 0}, (set) => ({
                    incremen: () =>
                        set(
                            (state) => {
                                state.count += 1;
                            },
                            false,
                            'INCREMENT',
                        ),
                    decrement: () => 
                        set(
                            (state) => {
                                state.count -= 1;
                            },
                            false,
                            'DECREMENT',
                        ),
                    reset: () => 
                        set(
                            (state) => {
                                state.count = 0;
                            },
                            false,
                            'RESET',
                        ),
                })),
            ),
            {
                name: 'counter-storage',
            },
        ),
    ),
);
```

### combine
- 여러 상태를 하나의 스토어로 합칠때 유용한 미들웨어
- 각 상태를 따로 관리하는 대신, 객체 형태로 병합하여 한번에 관리할 수 있도록 지원

```
export const useCombineCounterStore = create<CounterState>()(
    combine({count: 0}, (set) => ({
        increment: () => set((state) => ({count : state.count + 1})),
        decrement: () => set((state) => ({count : state.count - 1})),
        reset: () => set({count : 0}),
    })),
);
```

### immer
- 상태를 직접 변경할 수 있도록 도와주는 불변성 관리 미들웨어
```
export const useImmerCounterStore = create<CounterState>() {
    immer((set) => ({
        count: 0,
        increment : () =>
            set((state) => {
                state.count +=1;
            });
        decrement : () => 
            set((state) => {
                state.count -= 1;
            })
        reset: () => 
            set((state) => {
                state.count = 0;
            })
    })),
}
```

### subscribeWithSelector
- 상태의 변화를 감지하여, 이벤트 리스너를 등록하고 변경될 때 실행할 수 있도록 지원
```
export const useSubScribeCOunterStore = create<CounterState>()(
    subscribeWithSelector(
        immer((set) => ({
            count: 0,
            increment: () =>
                set((state) => {
                    state.count += 1;
                }),
            decrement : () => 
                set((state) => {
                    state.count -= 1;
                }),
            reset: () => 
                set((state) => {
                    state.count = 0;
                })
        })),
    ),
);

useSubScribeCounterStore.subscribe(
    (state) => state.count,
    (count) => {
        console.log('coun 변경됨', count);
    },
);

```
