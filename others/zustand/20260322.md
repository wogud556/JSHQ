## Zustand
- 독일어로 상태라는 뜻을 가진 라이브러리

### 특징
- 특정 라이브러리에 엮이지 않음(그래도 React와 함께 쓸 수 있는 API는 기본적으로 제공함)
- 한개의 중앙에 집중된 형식의 스토어 구조를 활용하면서 상태를 정의하고 사용하는 방법이 단순함
- Context API를 사용할 때와 달리 상태 변경 시 불필요한 리랜더링을 일으키지 않도록 제어하기 쉬움
- React에 직접적으로 의존하지 않기 때문에 자주 바뀌는 상태를 직접 제어할 수 있는 방법도 제공함(Transient Update)
- 동작을 이해하기 위해 알아야 하는 코드 양이 아주 적음. 핵심 로직의 코드 수가 약 42줄밖에 되지 않음(VanillaJS 기준)

### Context API를 사용하지 않은 이유
- Context는 컴포넌트에 의존성을 주입할 수 있는 아주 효과적인 방법 중 하나임
- 하지만 부모 컴포넌트 쪽엥 Context.Provider 컴포넌트를 선언하고 Context로 전달되는 값이 변경될 때 해당 Context를 사용하는 모든 자손 컴포넌트는 리랜더링 됨
- 다음과 같이 임의의 값과 값을 변경하는 방법을 제공하는 Context와 Provider를 만들었다고 가정해봄
```
const SomeObjectContext = React.createContext({input : '', count: 0});
const SetSomeObjectContext = React.createContext();

const Provider = ({ children } => {
    const [someObj, setSomeObj] = React.useState({input:'', count:0});

    return(
        <SetSomeObjectConteext.Provider value={setSomeObj}>
            <SomeObjectContext.Provider value = {someObj}>
                {children}
            </SomeObjectContext.Provider>
        </SetSomeObjectConteext.Provider>
    )
});
```
- 이 Context를 소비하는 컴포넌트가 여러 단계에서 아래쪽 자손일 수 있음

```
const InputConsumer = () => {
    const { input } = useContext(SomeObjectContext);
    //...
}

const CountConsumer = () => {
    const { count } = useContext(SomeObjectCOntext);
}

const App = () => {
    <Provider>
        <DeepChildren>
            <SomeOtherChidren>
                <AnotherChildren>
                    {}
                    <InputConsumer>
                </AnotherChildren>
                <CountConsumer />
            </SomeOtherChidren>
        </DeepChidren>
        <IDontCareContextChild />
        <ContextSetter />
    </Provider>
}
```

### 간략한 Zustand 사용 방법
- 스토어를 만들 때는 create 함수를 이용하여 상태와 그 상태를 변경하는 액션을 정의함 그러면 리액트 컴포넌트에서 사용할 수 있는 useState훅을 리턴함
```
import create from 'zustand';

// set 함수를 통해서만 상태를 변경할 수 있음
const useState = create(set => ({
    bear:0,
    increasePopulation : () => set(state => ({bears : state.bears + 1}))
    removeAllBears: () => set({bears:0})
}));
```
- 컴포넌트에서 useStore 훅을 사용할 때는 스토어에서 상태를 어떤 형태로 꺼내올지 결정하는 함수를 전달해 줘야 함
- 만약 셀렉터 함수를 전달하지 않으면 스토어 전체가 리턴됨

```
// 상태를 꺼냄
function BearCounter(){
    const bears = useState(state = state.bears);
    return <h1>{bears}around hear ... </h1>;
}

function Controls(){
    const increasePopulation = useState(state => state.increasePopulation);
    return <button onClick = {increasePopulation}>one up</button>;
    
}
```

### Zustand 파헤처보기 - 코어
- Zustand는 발행/구독 모델을 기반으로 이루어짐
- 스토어의 상태 변경이 일어날 때에 실행할 리스터 함수를 모아두었다가(구독) 상태가 변경되었을 때 등록된 리스너들에게 상태가 변경되었다고 알려줌
- 또한 스토어를 생성하는 함수를 호출할 때 클로저를 활용함
- 클로저는 간단히 말해 함수가 선언될 시 그 주변 환경을 기억하는 것
- 아래의 코드를 보면 스토어의 상태는 스토어를 조회하거나 변경하는 함수 바깥 스코프에 항상 유지되도록 만들어진 것을 볼 수 있음
- 그러면 상태 변경, 조회 구독 등의 인터페이스를 통해서만 스토어를 다루고 실제 상태는 애플리케이션의 생명 주기
